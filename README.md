## 1. ที่มาและความสำคัญ (Background & Significance)

ในยุคปัจจุบัน พฤติกรรมของผู้บริโภคได้เปลี่ยนแปลงไปอย่างพลิกโฉม จากการเดินเลือกซื้อสินค้าตามหน้าร้านในศูนย์การค้า ขยับขยายเข้าสู่แพลตฟอร์มออนไลน์อย่างเต็มรูปแบบ โดยเฉพาะอย่างยิ่งในอุตสาหกรรมแฟชั่นและเสื้อผ้าที่มีอัตราการแข่งขันสูงมาก ผู้บริโภคในยุคดิจิทัลไม่เพียงแต่ต้องการสินค้าที่มีคุณภาพและตรงตามเทรนด์เท่านั้น แต่ยังคาดหวังประสบการณ์การช็อปปิ้ง (Shopping Experience) ที่รวดเร็ว ใช้งานง่าย สะดวกสบาย และมีความน่าเชื่อถือสูงตั้งแต่ก้าวแรกที่เข้าสู่เว็บไซต์

อย่างไรก็ตาม เมื่อมองภาพรวมของแพลตฟอร์ม E-Commerce จำนวนมากในท้องตลาด จะพบว่าหลายแห่งยังคงเผชิญกับข้อจำกัดที่ส่งผลกระทบต่อความพึงพอใจของลูกค้าโดยตรง ไม่ว่าจะเป็นปัญหาหน้าเว็บไซต์ที่โหลดข้อมูลล่าช้าจนทำให้เกิดอัตราการตีกลับ (Bounce Rate) สูง ระบบการค้นหาที่ไม่แม่นยำและขาดฟังก์ชันการคัดกรอง (Filter) สี ไซส์ หรือสไตล์ที่ต้องการ ไปจนถึงขั้นตอนการชำระเงินที่ยุ่งยากซับซ้อนเกินความจำเป็น ซึ่งปัจจัยเหล่านี้ล้วนเป็นสาเหตุหลักที่ทำให้เกิดปัญหาการละทิ้งตะกร้าสินค้า (Cart Abandonment) กลางคันและทำให้ร้านค้าสูญเสียโอกาสในการขาย

นอกเหนือจากปัญหาในกระบวนการเลือกซื้อแล้ว อีกหนึ่งช่องโหว่สำคัญคือการขาดแคลนระบบบริหารจัดการปัญหาหลังการขายและการจัดการออเดอร์ที่เป็นรูปธรรม เมื่อลูกค้าต้องการติดตามสถานะพัสดุหรือแจ้งเคลมสินค้า มักจะต้องเผชิญกับการประสานงานที่ล่าช้าและไม่มีระบบรองรับที่ชัดเจน จากข้อจำกัดทั้งหมดที่กล่าวมา จึงเป็นที่มาของการริเริ่มและพัฒนาระบบ "SALA STORE" แพลตฟอร์ม E-Commerce ที่มุ่งเน้นการแก้ปัญหารอยรั่วเหล่านี้แบบครบวงจร เพื่อส่งมอบประสบการณ์ที่ดีที่สุดให้กับลูกค้า พร้อมทั้งสร้างระบบหลังบ้านที่แข็งแกร่งสำหรับผู้ดูแลและพนักงานในการจัดการคำสั่งซื้อ การจัดส่ง และการดูแลลูกค้าอย่างมีประสิทธิภาพ

## 2. ปัญหาที่พบในปัจจุบัน (Pain Points)

###  ฝั่งลูกค้า (Customer)
| ปัญหา | ผลกระทบ | สาเหตุ |
| :--- | :--- | :--- |
| **ค้นหาสินค้าไม่เจอ** | ลูกค้าเสียเวลา เกิดความหงุดหงิด และปิดเว็บไซต์ | ไม่มีตัวกรอง (Filter) ตามไซส์, สี หรือราคา |
| **ชำระเงินซับซ้อน** | อัตราการทิ้งตะกร้าสินค้าสูง | ฟอร์มกรอกข้อมูลยาว ไม่มีการบันทึกที่อยู่ล่วงหน้า |
| **ดูประวัติไม่ได้** | ติดตามสถานะพัสดุหรืออ้างอิงเพื่อเคลมสินค้าไม่ได้ | ขาดระบบประวัติการสั่งซื้อ (Order History) |
| **แจ้งเรื่องยาก** | ต้องทักแชตเพจที่ตอบล่าช้า | ขาดช่องทางและระบบส่งคำร้อง (Ticket Support) |
| **ไม่มีการแจ้งเตือน** | ไม่ทราบความคืบหน้าของคำสั่งซื้อ | ขาดระบบ Notification อัปเดตสถานะ |

###  ฝั่งพนักงาน (Staff)
| ปัญหา | ผลกระทบ | สาเหตุ |
| :--- | :--- | :--- |
| **ตรวจสอบออเดอร์ยาก** | จัดเตรียมสินค้าผิดพลาดและล่าช้า | ขาดระบบคัดกรองสถานะคำสั่งซื้อของลูกค้า |
| **อัปเดตสถานะขนส่งช้า** | ลูกค้าทักมาตามของบ่อย สร้างภาระงานเพิ่ม | ไม่มีระบบจัดการสถานะคำสั่งซื้อและการจัดส่งที่ใช้งานง่าย |
| **ไม่มีระบบจัดการเคส** | สับสนว่าเคสไหนแก้ไขแล้ว หรือยังค้างอยู่ | ใช้แชตรวมส่งผลให้ไม่มีการบันทึกสถานะของแต่ละปัญหา |
| **เข้าไม่ถึงข้อมูลการขาย** | ประเมินหน้าร้านและการเติมสต็อกเบื้องต้นไม่ได้ | ไม่มีสิทธิ์เข้าดูข้อมูลสรุปยอดขายประจำวัน |

###  ฝั่งผู้ดูแลระบบ (Admin)
| ปัญหา | ผลกระทบ | สาเหตุ |
| :--- | :--- | :--- |
| **ไม่มีจุดศูนย์กลาง** | ต้องสลับหลายระบบ ทำงานล่าช้า | ขาดศูนย์รวมข้อมูลหลังบ้าน (Dashboard) |
| **จัดการสต็อกยาก** | อัปเดตสต็อกช้า เกิดปัญหา Overselling | ไม่มีระบบจัดการสินค้า (CRUD) ที่เชื่อมต่อเรียลไทม์ |
| **จัดหมวดหมู่สับสน** | สินค้าจัดวางสับสน ลูกค้าหาของไม่เจอ | ขาดระบบ Category Management ที่ยืดหยุ่น |
| **มองไม่เห็นภาพรวม** | วางกลยุทธ์ช้า ขาดข้อมูลในการตัดสินใจ | ขาดระบบสรุปสถิติด้านยอดขายเชิงลึกและรายงานผู้ใช้งาน |
| **ควบคุมสิทธิ์ไม่ได้** | พนักงานเข้าถึงข้อมูลส่วนที่ไม่จำเป็น เสี่ยงต่อความปลอดภัย | ขาดระบบการจัดการสิทธิ์ (Role Assignment) |

## 3. วัตถุประสงค์ (Objectives)
1. พัฒนาเว็บไซต์ E-Commerce แบรนด์ SALA ที่แสดงผลได้สวยงามและรองรับ **Responsive Design**
2. สร้างระบบการซื้อขาย ตะกร้าสินค้า และประวัติการสั่งซื้อที่มีประสิทธิภาพ ปลอดภัย
3. พัฒนาระบบ Customer Support ในรูปแบบ **Ticket** และระบบจัดการหลังบ้านที่เชื่อมต่อพนักงานและลูกค้าอย่างไร้รอยต่อ
4. สร้าง **Admin & Staff Dashboard** ที่ช่วยบริหารจัดการสินค้า หมวดหมู่ คำสั่งซื้อ และสิทธิ์ผู้ใช้งานได้ในจุดเดียว
5. ออกแบบสถาปัตยกรรมระบบที่เสถียร รองรับการขยายตัว (**Scalable Architecture**) ในอนาคต

## 4. ขอบเขตของระบบ (Scope of the System)

###  สิ่งที่อยู่ในขอบเขต (In Scope)
- ระบบ Authentication & Profile (สมัครสมาชิก, เข้าสู่ระบบ, จัดการโปรไฟล์)
- ระบบแสดงผล ค้นหา และคัดกรองสินค้า (หมวดหมู่, ไซส์, สี, ราคา)
- ระบบ Cart & Checkout (ตะกร้าสินค้าและการสั่งซื้อ)
- ระบบชำระเงิน (เชื่อมต่อ Payment Gateway)
- ระบบบันทึกและแสดงประวัติการสั่งซื้อและติดตามสถานะพัสดุ
- ระบบ Ticket Support System (สำหรับยื่นข้อร้องเรียนและตอบกลับ)
- Staff Workspace (ระบบตรวจสอบคำสั่งซื้อ, อัปเดตการจัดส่ง, ดูยอดขายเบื้องต้น, และตอบ Ticket)
- Admin Dashboard (จัดการสินค้า, หมวดหมู่, สิทธิ์ผู้ใช้งาน, และรายงานการขายเชิงลึก)
- ระบบแจ้งเตือนทางอีเมล (Notification พื้นฐาน)

###  สิ่งที่อยู่นอกขอบเขต (Out of Scope)
- ระบบแนะนำเสื้อผ้าด้วย AI (Recommendation Engine)
- แอปพลิเคชัน Mobile Native (iOS / Android)
- ระบบ Live Chat แบบเรียลไทม์
- ระบบสะสมแต้ม (Loyalty Points)
- ระบบ Marketplace สำหรับผู้ขายหลายราย

## 5. บทบาทผู้ใช้งาน (User Roles)
1. **Customer (ลูกค้า):** ผู้ใช้งานทั่วไปที่เลือกชม ค้นหา สั่งซื้อ ชำระเงิน ตรวจสอบสถานะ/ประวัติ และส่ง Ticket แจ้งปัญหา 
2. **Staff (พนักงาน):** พนักงานผู้ดูแลระบบส่วนหน้า มีหน้าที่ตรวจสอบคำสั่งซื้อ อัปเดตสถานะการจัดส่ง ดูข้อมูลยอดขายเบื้องต้น และตอบกลับ Ticket เพื่อช่วยเหลือลูกค้า
3. **Admin (ผู้ดูแลระบบ):** ผู้ดูแลสูงสุด มีสิทธิ์จัดการข้อมูลสินค้า หมวดหมู่ ตรวจสอบบัญชีผู้ใช้งาน กำหนดสิทธิ์การใช้งาน (Role) และดูรายงานสถิติยอดขายเชิงลึกทั้งหมด

## 6. ความต้องการของผู้ใช้งาน (User Requirements)

###  Customer
* **Log in / Register:** สมัครสมาชิกและเข้าสู่ระบบเพื่อใช้งานระบบอย่างปลอดภัย
* **Search & Browse Product:** ค้นหาพร้อมตัวกรอง (หมวดหมู่, ไซส์, สี, ราคา) และดูรายละเอียดสินค้า
* **Cart Management:** เพิ่ม ลบ หรือแก้ไขจำนวนสินค้าในตะกร้าได้ตามต้องการ
* **Checkout & Payment:** สั่งซื้อสินค้า กรอกที่อยู่ เลือกวิธีชำระเงิน และตรวจสอบยอดรวม
* **Order Tracking & History:** ดูประวัติการสั่งซื้อย้อนหลังและติดตามสถานะพัสดุได้จริง
* **Submit Ticket:** เปิดเคสแจ้งปัญหา แนบรูป และติดตามการตอบกลับจากพนักงาน

###  Staff
* **View Orders:** เข้าถึงและตรวจสอบรายละเอียดคำสั่งซื้อของลูกค้า
* **Update Order & Shipping Status:** อัปเดตสถานะของคำสั่งซื้อและข้อมูลการจัดส่งพัสดุ
* **View Basic Sales:** ตรวจสอบข้อมูลการขายและสถิติยอดขายเบื้องต้นประจำวันเพื่อเตรียมสต็อก
* **Reply Ticket:** ตรวจสอบรายการคำร้อง คัดกรองสถานะ พิมพ์ข้อความช่วยเหลือลูกค้า และอัปเดตปิดเคส

###  Admin
* **Product CRUD:** จัดการข้อมูลเสื้อผ้า รูปภาพ ราคา ไซส์ สี และสต็อก (เพิ่ม/แก้ไข/ลบ)
* **Overall Order Management:** ตรวจสอบและจัดการคำสั่งซื้อทั้งหมดในภาพรวมของระบบ
* **User Management:** ดูรายชื่อ ค้นหา และจัดการข้อมูลบัญชีผู้ใช้งานทั้งหมด
* **Role Assignment:** กำหนดและปรับเปลี่ยนสิทธิ์การใช้งานระบบให้กับผู้ใช้หรือพนักงาน (Extend จากระบบจัดการผู้ใช้)
* **Advanced Sales Report:** ตรวจสอบรายงานและสถิติยอดขายเชิงลึกเชิงวิเคราะห์ของร้านค้า

## 7. ความต้องการของระบบ (System Requirements)

### Functional Requirements (FR)
| โมดูล | รายละเอียด |
| :--- | :--- |
| **Authentication** | รองรับการสมัคร, ล็อกอิน, รีเซ็ตรหัสผ่าน, Password Hashing ด้วย bcrypt |
| **Catalog & Search** | แสดงสินค้า ค้นหาด้วยคีย์เวิร์ด กรองและจัดเรียงข้อมูลตามไซส์/สี/ราคา |
| **Cart & Checkout** | คำนวณราคารวม ภาษี บันทึกที่อยู่ และตัดสต็อกสินค้าในตาราง Variant เมื่อชำระเงินสำเร็จ |
| **Payment Integration**| เชื่อมต่อ Gateway รองรับการตัดจ่ายผ่านบัตรเครดิต, พร้อมเพย์, และ TrueMoney |
| **Order Tracking** | บันทึกประวัติและแสดงสถานะออเดอร์ พร้อมระบบให้ Staff เข้าไปอัปเดตสถานะจัดส่งได้ |
| **Ticket System** | สร้างฟอร์มผูกกับ Order ID และมีหน้าจัดการเคส (Workspace) ให้สำหรับทีม Staff ตอบคำร้อง |
| **Management Dashboard**| อินเทอร์เฟซแยกตามสิทธิ์: Staff ดูยอดขายเบื้องต้น/ออเดอร์; Admin จัดการสินค้า/สิทธิ์/รายงานเชิงลึก |
| **Notification** | ส่งอีเมลแจ้งเตือนอัตโนมัติเมื่อออเดอร์สำเร็จ หรือได้รับการอัปเดตสถานะ Ticket |

### Non-Functional Requirements (NFR)
| หมวดหมู่ | รายละเอียด |
| :--- | :--- |
| **Performance** | โหลดข้อมูลหน้าร้านเสร็จสิ้นและพร้อมใช้งานภายในเวลา **2 วินาที** |

| **Scalability** | ออกแบบโครงสร้างระบบแยกส่วน (Microservice-ready) รองรับ Traffic จำนวนมากช่วงเทศกาล |
| **Usability** | การดีไซน์สไตล์มินิมอล ใช้งานง่ายไม่ซับซ้อน และรองรับรูปแบบ **Responsive** บนทุกขนาดหน้าจอ |
| **Reliability** | ระบบหลังบ้านและหน้าบ้านต้องมีอัตรา Uptime ไม่ต่ำกว่า **99.5%** ต่อเดือน |

## 8. เทคโนโลยีที่ใช้พัฒนา (Technology Stack)

* **Frontend:**
  * `React.js` - สำหรับสร้าง User Interface แบบ Component-based (Single Page Application)
  * `Tailwind CSS` - จัดสไตล์หน้าเว็บแนว Minimal แบบ Utility-First สะดวกรวดเร็ว
  * `Bootstrap` - ใช้ Grid System เพื่อช่วยจัดการ Layout และโครงสร้างให้เสถียรทุกหน้าจอ
* **Backend:**
  * `Node.js (Express.js)` - สร้าง RESTful API เพื่อประมวลผลข้อมูลที่รวดเร็วแบบ Non-blocking I/O
* **Database:**
  * `MySQL` - ระบบฐานข้อมูลเชิงสัมพันธ์ (Relational Database) รองรับ ACID Transaction ป้องกันปัญหาสต็อกรวน
* **Tools:**
  * `GitHub` - ใช้สำหรับจัดเก็บ Source Code และจัดการระบบ Version Control ภายในกลุ่ม
  * `Sourcetree` - โปรแกรม Git GUI Client ช่วยในการ Visualize และบริหารจัดการ Branch / Merge Code

## 9. โครงสร้างฐานข้อมูล (Database Design)

ตารางหลัก (Core Tables) ในระบบ MySQL ประกอบด้วย:
* `users` - เก็บข้อมูลผู้ใช้และแบ่งสิทธิ์โดยคอลัมน์ role (`customer`, `admin`, `staff`)
* `addresses` - ที่อยู่สำหรับจัดส่งสินค้า (ความสัมพันธ์แบบ 1 User : N Addresses)
* `categories` - หมวดหมู่ของเสื้อผ้าแฟชั่น
* `products` - ข้อมูลสินค้ารุ่นหลัก (เช่น ชื่อสินค้า, รายละเอียดสินค้า)
* `product_variants` - ตัวเลือกสินค้าย่อยที่ลงลึก (แยกตาม ไซส์ / สี / และสต็อกคงเหลือ)
* `carts` - ข้อมูลตะกร้าสินค้าหลักของผู้ใช้งานแต่ละคน
* `cart_items` - รายการสินค้าและจำนวนที่อยู่ในตะกร้า
* `orders` - ข้อมูลคำสั่งซื้อหลักและสถานะรวมของคำสั่งซื้อนั้นๆ
* `order_items` - รายการสินค้าที่ถูกสั่งซื้อจริงในออเดอร์นั้น
* `payments` - ข้อมูลประวัติการชำระเงินและเลขอ้างอิงจาก Gateway
* `tickets` - คำร้องหรือปัญหาที่ส่งเข้ามา (เชื่อมโยงผูกกับ User ID และ Order ID)
* `ticket_messages` - กล่องข้อความบันทึกการสนทนาระหว่าง Customer และ Staff ในแต่ละ Ticket
